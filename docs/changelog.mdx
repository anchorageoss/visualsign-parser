## How releases are tagged

Tags are auto-cut from `main` by the release workflow. The `description` on each `<Update>` block below points to the first release that contains the change. If you pull a tag at or above that version, the change is in.

## How entries are authored

Contributors drop a draft fragment in `docs/changelog/_drafts/` on the PR that ships the change. A maintainer batches drafts into one or more `<Update>` blocks at curation time, splits them across the chain-parsers and fundamentals pages by category, and deletes the consumed drafts. See [`docs/changelog/_drafts/README.md`](https://github.com/anchorageoss/visualsign-parser/blob/main/docs/changelog/_drafts/README.md) for the format.

## Tag vocabulary

Both streams use a shared controlled tag vocabulary so RSS subscribers can filter:

- **Audience** — `Wallet API`, `Contributors`, `Security`, `Ecosystem`.
- **Chain** — `Ethereum`, `Solana`, `Sui`, `Tron` (chain parsers only).
- **Theme** — `Policy`, `Architecture`, `Lints`, `Attestation`, `Performance`.
